Android

The Android build doesn't need this site's packager at all -- it imports your disc's game content itself, on-device, the first time you open it. This page is just the APK link and the exact steps.

1. Get the APK

Grab the latest arm64 APK from the project's GitHub Releases page (look for the .apk asset, roughly 30MB). It's a clean build: no game code or assets are in it, same policy as the Windows build -- see the legal section on the home page.

Your device will warn you about installing from outside the Play Store ("unknown sources") -- that's expected for a small unsigned/independently-signed fan project, not a sign anything's wrong. Enable it for the browser or file manager you're installing from, install the APK, then you can turn that setting back off.

2. Open the app and select your disc image

On first launch the app shows a Game Content Setup screen -- it looks for Mario Party 6 content and, not finding any yet, offers two buttons: Select Disc Image and Select Folder.

  • Select Disc Image opens the system file picker -- point it at a .iso/.gcm (or another disc-image format your dump happens to be in) copied onto your device or reachable through it (cloud storage, a USB drive, etc).
  • Select Folder opens Android's document-tree picker -- use this if you already have the disc extracted to a folder (Dolphin's "Extract Entire Disc...") rather than as a single image file.

Either way, the app is doing the same job this site's packager does for Windows, just on-device instead of in a browser tab: it verifies the disc is really Mario Party 6 (USA), then imports the roughly 400MB subset of files the port actually needs (everything under data/, mess/ and mic/, plus the opening banner and the two streamed-audio containers -- movies and the rest are skipped, they're not needed past the menu). A progress bar tracks bytes and file count as it copies.

3. Play

Once the import finishes, the content is kept on your device (under the app's own storage area) and every later launch finds it automatically -- you only do this once per disc. As with the Windows build, gameplay currently stops at the mode-select menu; see the project's README for what's implemented so far.

Notes

  • Everything above runs entirely on your device. Nothing about your disc or its contents is ever sent anywhere.
  • The imported content lives in the app's own storage; uninstalling the app removes it too (you'd import again from your disc after reinstalling).
  • Please don't rehost or redistribute the imported content -- see the legal section for why.
